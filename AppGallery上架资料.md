## 📝 应用描述

### 一句话简介（30字以内）
精准的花粉浓度预报，过敏患者的贴心助手

### 详细描述（300-1000字）

**花粉浓度播报** 是一款专为花粉过敏人群打造的鸿蒙原生应用，提供精准的花粉浓度预报和健康防护建议。

**核心功能**：

🌸 **实时花粉监测**
- 支持全国主要城市的实时花粉浓度查询
- 整合 Google Pollen API 和气象局数据
- 显示花粉类型、浓度等级、过敏风险指数

📊 **5日预报趋势**
- 未来5天花粉浓度变化趋势
- 直观的折线图展示，提前规划出行
- 温度、湿度、风速等相关天气信息

📍 **精准定位服务**
- GPS 自动定位，获取当前位置花粉信息
- 支持 23+ 重点城市手动切换
- 城市收藏与置顶功能，快速查看常用城市

🗺️ **花粉地图可视化**
- 基于华为 Map Kit 的花粉分布地图
- 地图标记显示当前位置花粉等级
- 毛玻璃信息卡片展示实时数据

⚙️ **个性化设置**
- 震动反馈开关
- 深色模式 / 浅色模式切换
- 权限管理与隐私控制

**适用人群**：
- 花粉过敏患者（中国约2亿人群）
- 有户外活动需求的健康意识人群  
- 需要照顾过敏家庭成员的用户

**技术亮点**：
- 鸿蒙 API 20 原生开发
- ArkTS + ArkUI 现代化界面
- 支持服务卡片（桌面小组件）
- 数据本地缓存，离线也能查看历史记录

## 📄 隐私政策

### 权限说明

**必需权限**：

1. **网络访问权限** (`ohos.permission.INTERNET`)
   - 用途: 获取花粉浓度和天气数据
   - 使用场景: 启动时、用户刷新数据时

2. **位置权限** (`ohos.permission.LOCATION` + `ohos.permission.APPROXIMATELY_LOCATION`)
   - 用途: 自动获取用户所在城市，提供精准花粉预报
   - 使用场景: 用户点击"定位"按钮时
   - 隐私承诺: 仅用于获取城市信息，不记录用户行踪

3. **震动权限** (`ohos.permission.VIBRATE`)
   - 用途: 操作反馈（可在设置中关闭）
   - 使用场景: 用户点击按钮时

### 数据收集说明

**我们收集的数据**：
- 用户选择的城市列表（本地存储）
- 应用偏好设置（震动开关、外观模式）

**我们不收集的数据**：
- 不收集用户个人身份信息
- 不追踪用户位置轨迹
- 不上传数据到第三方服务器

**数据存储**：
- 所有数据均存储在本地设备
- 使用 AppStorage 和 PersistentStorage 本地化存储
- 卸载应用时数据自动清除

### 第三方服务

**Google Pollen API** (通过 AWS Lambda 代理)
- 用途: 获取全球花粉浓度数据
- 数据传输: 仅传输经纬度坐标，不包含用户标识
- 隐私政策: https://policies.google.com/privacy

**华为 Map Kit**
- 用途: 地图显示
- 数据传输: 地图瓦片加载，符合华为隐私政策
- 隐私政策: https://developer.huawei.com/consumer/cn/doc/

## 📦 HAP 包生成步骤

### 1. 清理构建缓存
```bash
# 在 DevEco Studio Terminal 执行
hvigorw clean
```

### 2. 检查签名配置
确认 `build-profile.json5` 中的签名配置正确：
- ✅ `storeFile`: 证书文件路径存在
- ✅ `keyAlias`: 与证书匹配
- ✅ `profile`: 发布证书文件存在
- ✅ `certpath`: 证书文件存在

### 3. 构建 Release HAP
```bash
# DevEco Studio 菜单操作
Build → Build Hap(s)/APP(s) → Build Hap(s)
# 或使用命令行
hvigorw assembleHap --mode module -p module=entry@default -p product=default
```

### 4. 验证 HAP 包
输出路径: `entry/build/default/outputs/default/entry-default-signed.hap`

检查项:
- [ ] 文件大小合理（预计 5-15MB）
- [ ] 使用正式签名（不是 debug 签名）
- [ ] 包名正确: com.eric.PollenForecast
