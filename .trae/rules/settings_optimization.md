# TRAE_CN IDE 设置优化指南

根据您的反馈，以下是针对MCP和智能体设置的优化建议，帮助您关闭不必要的功能以提高IDE性能和使用体验。

## 🎯 优化原则

- **按需启用**：只启用与当前HarmonyOS项目开发相关的功能
- **减少负担**：关闭不必要的自动运行和通知功能
- **简化界面**：减少工具栏和菜单中的冗余选项

## 🔧 MCP (Multi-Cloud Platform) 设置优化

### 当前状态分析
根据您提供的截图，MCP列表中显示多个服务器：
- harmonyos-native-app-examples (已启用)
- local-database-fixed (启动失败)
- harmonyos-docs-server (启动失败)
- software-planning-tool (已启用)
- SequentialThinking (已启用)
- spec-workflow (已启用)
- Memory (已启用)

### 优化建议
1. **关闭启动失败的服务器**：
   - 点击 "local-database-fixed" 和 "harmonyos-docs-server" 右侧的开关按钮
   - 这些失败的服务器会占用资源且无实际功能

2. **精简必要服务器**：
   - 仅保留与HarmonyOS开发相关的服务器：`harmonyos-native-app-examples`
   - 根据实际需求选择启用其他服务器，如：
     - 仅在需要文档服务时启用 `software-planning-tool`
     - 仅在需要特殊工作流时启用 `spec-workflow`

3. **禁用自动运行MCP**：
   - 点击左侧菜单栏 → `设置` → `对话流`
   - 关闭 "自动运行MCP" 开关
   - 需要时手动在MCP面板中启用

## 🤖 智能体设置优化

### 当前状态分析
- 内置智能体：Chat、Builder、Builder with MCP、SOLO Coder
- 自定义智能体：量化分析师、原生鸿蒙产品经理
- 所有智能体默认处于可访问状态

### 优化建议
1. **保留核心智能体**：
   - 保留 `Builder with MCP` 和 `SOLO Coder` (3.0.0 Only)
   - 其他智能体（如Chat、Builder）可根据需求切换

2. **禁用自动运行智能体**：
   - 点击左侧菜单栏 → `智能体`
   - 检查并关闭不需要的智能体
   - 只在需要特定功能时启用

3. **关闭自动修复和自动接受变更**：
   - 点击左侧菜单栏 → `设置` → `对话流`
   - 关闭 "自动修复" 和 "自动接受文件变更" 开关
   - 手动控制代码变更，确保代码质量

## ✨ 对话流设置优化

### 优化建议
1. **关闭自动运行命令**：
   - 点击左侧菜单栏 → `设置` → `对话流`
   - 关闭 "自动运行命令" 开关
   - 或调整为 "使用白名单" 模式，只允许必要命令

2. **简化任务状态通知**：
   - 关闭 "声音" 通知，保留 "横幅" 通知
   - 降低音量设置（如调整到50%）

3. **关闭对话流自动折叠**：
   - 关闭 "对话流点击启动折叠" 开关
   - 保持对话历史完整可见

## 🎨 CUE (Context Understanding Engine) 设置优化

### 当前状态分析
- Tab-CUE (已启用)
- 智能导入 (已启用)
- 智能重命名 (已启用)

### 优化建议
1. **按需启用智能功能**：
   - 保留 "Tab-CUE" 以获得基本的上下文理解
   - 智能导入和智能重命名目前处于Beta阶段，可根据使用体验选择启用/禁用

## 📊 性能优化总结

| 功能类别 | 优化建议 | 预期效果 |
|---------|---------|---------|
| MCP服务器 | 关闭失败和不必要的服务器 | 减少资源占用，提高IDE响应速度 |
| 智能体 | 只保留核心智能体 | 简化界面，减少后台进程 |
| 自动功能 | 关闭自动修复、自动接受变更、自动运行命令 | 手动控制开发流程，避免意外修改 |
| 通知 | 简化通知设置 | 减少干扰，提高专注度 |

## 🚀 应用优化设置后

1. 重启TRAE_CN IDE以应用所有更改
2. 观察IDE性能变化，如响应速度和内存占用
3. 根据实际开发需求，可逐步调整设置以找到最佳配置

## 💡 额外建议

- **定期清理**：每月检查一次设置，关闭不再使用的功能
- **更新版本**：保持TRAE_CN IDE为最新版本，以获得更好的性能优化
- **反馈问题**：如遇到性能问题，可通过TRAE_CN的反馈渠道提交报告

以上优化建议基于您的HarmonyOS花粉浓度播报项目需求，您可以根据个人使用习惯进行调整。
