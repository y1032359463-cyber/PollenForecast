# "到时间了吗" App 最终页面设计方案

## 页面流程图
```
首页(身份选择)
    │
    ├─→ 老板端
    │   ├─ 我的业务(主页)
    │   ├─ 业务管理页(资源列表)
    │   ├─ 收款码页面(纯二维码)
    │   └─ 历史记录
    │
    └─→ 消费者端
        ├─ 我的租赁(主页)
        ├─ 选择业务类型
        ├─ 资源列表
        ├─ 租赁详情
        └─ 历史记录
```

## 核心定位
- **纯计时工具** - 主要功能是倒计时
- **计费仅供参考** - 实际收费老板自己定
- **固定收款码** - 不需要动态生成金额
- **极简操作** - 越简单越好
- **左侧固定分类栏** - 业务类型永久显示在左侧,点击或拖拽添加,最多4个

## 核心功能总结

**老板端流程:**
**老板端流程:**
```
1. 左侧拖拽业务类型到主页(首次使用)
2. 点击业务卡片 → 进入业务管理页
3. 点击空闲资源 → 选择倒计时时间(1h/2h/3h/自定义)
4. 开始倒计时
5. 点击[查看收款码] → 显示纯收款码页面
6. 客户扫码付款
7. 返回业务管理页 → 点击停止 → 自动记录历史
```

**消费者端流程(主动租赁):**
```
1. 打开App → 点击[+租赁]
2. 选择业务类型(桌球/充电宝/单车/仓库等)
3. 选择商家和可用资源
4. 确认租赁 → 开始计时
5. 查看实时已使用时间和费用
6. 设置时间提醒
7. 点击[归还完成]结束租赁
```
**核心特点:**
1. **左侧固定分类栏** - 业务类型永久显示,点击或拖拽添加,最多4个
2. **倒计时功能** - 开始前选择倒计时,主页实时同步
3. **纯计时工具** - 计费仅供参考,实际收费老板自己定
4. **极简收款码页** - 只显示两个大二维码,上绿下蓝
5. **固定收款码** - 不动态生成金额,直接展示收款码图片
6. **自动记录历史** - 停止后自动保存到本地

**MVP极简版 - 砍掉的功能:**
- ❌ 数据统计(饼图/TOP排行等)
- ❌ 设置页面(收款码/计费规则)
- ❌ 消费者端
- ✅ 只保留核心: 计时 + 收款码展示 + 历史记录

**设计原则:**
- 越简单越好
- 减少多余步骤
- 二维码尽可能大
- 操作流畅不繁琐
- 收款码管理(上传固定收款码)
- 业务类型/资源/计费规则配置
- 可切换到消费者端

## 核心功能总结

**老板端流程:**
