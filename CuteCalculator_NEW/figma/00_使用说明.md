# Figma 设计提示词使用说明

## 简介
本目录包含用于 Figma Make AI 的设计提示词，用于生成 CuteCalculator 的 UI 设计稿。

**重要**: 所有提示词遵循统一的设计系统，参考 `globals.css` 中定义的变量。

## 文件列表
- **00_使用说明.md** - 本文件
- **globals.css** - 设计系统变量定义
- **Guidelines.md** - 完整设计规范和提示词模板
- **01_主计算器界面.txt** - 主界面设计提示词（待生成）
- **02_数字按钮素材.txt** - 数字按钮(0-9)设计提示词（已更新）
- **03_运算符按钮素材.txt** - 运算符按钮(+,-,×,÷)设计提示词（已更新）
- **04_功能按钮素材.txt** - 功能按钮(=,C,DEL,%)设计提示词（已更新）
- **05_卡通角色素材.txt** - 角色(猫咪4种状态×2层)设计提示词（已更新）

## 📋 核心设计原则

### 设计风格
- **主题**: 可爱、温暖、友好
- **配色**: 橙粉色系 (#FF8C42 橙色 + #FFB3C1 粉色)
- **背景**: 奶油色 (#FFF5E6)
- **字体**: HarmonyOS Sans (SF Pro Rounded 作为替代)
- **风格参考**: 日本 kawaii 风格、卡通可爱

### 关键特性
1. **角色动画**: 卡通猫咪角色，手臂可动画
2. **按钮样式**: 圆润、有阴影、渐变背景
3. **配色温暖**: 橙粉色调为主，奶油色背景
4. **触感友好**: 大按钮、高对比度、清晰层次

## 使用流程

### 第一步: 阅读设计规范
**必读**: 打开 `Guidelines.md` 了解完整的设计系统，包括：
- 颜色系统（5个主色 + 中性色）
- 字体规范（6个层级）
- 间距系统（8px网格）
- 圆角规范
- 阴影效果
- 动画参数

### 第二步: 复制通用设计上下文
每次使用提示词前，先复制 `Guidelines.md` 中的 **"Design System Context"** 部分。

这确保所有设计使用统一的：
- 颜色代码
- 字体大小
- 间距标准
- 圆角尺寸

### 第三步: 复制具体提示词
根据需要选择对应的 `.txt` 文件中的提示词：

**页面级提示词**:
- `01_主计算器界面.txt` - 完整的计算器主界面

**组件级提示词**:
- `02_数字按钮素材.txt` - 10个数字按钮（0-9）
- `03_运算符按钮素材.txt` - 4个运算符（+,-,×,÷）
- `04_功能按钮素材.txt` - 5个功能键（=,C,DEL,%,.）
- `05_卡通角色素材.txt` - 8个角色分层图片

### 第四步: 在 Figma 中生成
1. 打开 Figma (figma.com)
2. 创建新文件或打开现有文件
3. 按下 `Cmd/Ctrl + /` 打开 **Figma Make AI**
4. 粘贴：**设计上下文 + 具体提示词**
5. 点击生成

### 第五步: 调整细节
生成后检查：
- ✅ 颜色是否精确匹配（#FF8C42, #FFB3C1, #FFF5E6）
- ✅ 尺寸是否符合 @3x 规格（200×200px）
- ✅ 圆角是否为 48px (@3x)
- ✅ 阴影效果是否正确
- ✅ 文字对齐和间距

### 第六步: 分享设计给 AI 评估
1. 点击 Figma 右上角 "Share"
2. 设置为 "Anyone with the link can view"
3. 复制链接
4. 在 VS Code 中告诉 AI：
   > "请查看我的 Figma 设计: [粘贴链接]"

AI 会分析：
- 设计是否符合规范
- 尺寸、颜色、间距是否正确
- 是否需要调整提示词

### 第七步: 导出素材
确认设计无误后：

1. **选中要导出的元素**
2. **右侧面板 → Export 区域**
3. **设置导出参数**:
   - 格式: PNG
   - 尺寸: @3x (确保)
   - 背景: 透明（按钮和角色素材）
4. **点击 Export 导出**

**特别注意**: 
- 角色素材需要**分层导出**（身体层和手臂层分开）
- 数字 0 按钮是双倍宽度（416×200px @3x）
- 等号按钮是双倍高度（200×416px @3x）

### 第八步: 放入项目
将导出的 PNG 文件复制到:

### 第九步: AI 生成/优化代码
素材准备完成后，告诉 AI:
> "素材已准备完成，请验证项目是否可以编译运行"

AI 会：
1. 检查所有素材路径引用
2. 验证文件命名是否正确
3. 运行编译测试
4. 提供优化建议

## ⚠️ 重要注意事项

### ✅ 必须遵守
1. **颜色精确**: 使用精确的 HEX 代码，不要用近似色
2. **尺寸倍率**: 所有素材必须 @3x 导出（600px, 900px 等）
3. **透明背景**: 按钮和角色素材必须透明背景
4. **文件命名**: 严格按照命名规范（btn_0.png, cat_normal_body.png）
5. **分层导出**: 角色素材必须身体和手臂分开导出

### ⚠️ 常见错误
- ❌ 颜色使用 "orange" 而非 "#FF8C42"
- ❌ 导出 @2x 而非 @3x
- ❌ 角色只导出一层而非两层
- ❌ 文件命名不规范（button_0.png 而非 btn_0.png）
- ❌ 按钮有白色背景而非透明

### 💡 优化建议
1. **批量导出**: Figma 可以一次选中多个图层批量导出
2. **使用组件**: 创建按钮组件可以快速生成多个变体
3. **颜色样式**: 在 Figma 中创建颜色样式库确保一致性
4. **图层命名**: Figma 图层名直接影响导出文件名

## ❓ 常见问题

**Q: 为什么需要 @3x 尺寸？**  
A: HarmonyOS 使用 vp（虚拟像素）单位。@3x 确保在高分辨率屏幕上清晰显示。200×200px @3x = 66.7×66.7vp。

**Q: Figma Make AI 不理解中文怎么办？**  
A: 使用提供的英文提示词。如需调整，告诉我中文需求，我会生成英文提示词。

**Q: AI 如何读取我的 Figma 文件？**  
A: 分享 Figma Make 链接后，AI 可以分析文件内容、评估设计是否符合规范。

**Q: 角色分层导出怎么操作？**  
A: 在 Figma 中，身体层和手臂层应该在不同的 Frame 或 Group 中。分别选中后导出。

**Q: 可以手绘素材而不用 Figma 吗？**  
A: 可以！只要最终 PNG 文件符合尺寸、命名、透明背景等规范即可。

**Q: 素材不满意可以重新生成吗？**  
A: 完全可以！告诉 AI 哪里不满意，AI 会优化提示词，你重新在 Figma 中生成。
