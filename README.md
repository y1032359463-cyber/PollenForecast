# HarmonyOS 应用工厂项目

> 基于HarmonyOS 5.0的配置驱动应用开发方案 | Configuration-Driven HarmonyOS App Factory

## 📋 项目概述

本项目采用"应用工厂"模式，通过技术改造将华为 `Healthy_life` 示例应用转化为可配置的应用模板，实现70%+代码复用率的批量应用开发。

**核心目标：**
- 4周内开发8款原生鸿蒙应用
- 满足AppGallery审核标准（10+功能模块）
- 单个应用开发周期：7-10天
- 代码复用率：70%+

## 🛠 技术栈

- **平台**: HarmonyOS 5.0.0+ (API 12+)
- **IDE**: DevEco Studio 5.0.0+
- **语言**: ArkTS (TypeScript扩展)
- **架构**: MVVM + 配置驱动
- **数据库**: RelationalStore (RDB)
- **UI框架**: ArkUI组件化开发
- **设计**: Figma (AI/Make模式)

## 🚀 快速开始

### 环境要求
- Windows 10/11 或 macOS 10.14+
- DevEco Studio 5.0.0+
- Node.js 16+ (用于构建工具)
- Git 2.0+

### 克隆项目
```bash
git clone https://github.com/y1032959463-cyber/HarmonyOS_App_Plans.git
cd HarmonyOS_App_Plans
```

### 开发流程
1. **创建分支**: `git checkout -b feature/your-feature-name`
2. **开发功能**: 在DevEco Studio中编码
3. **提交代码**: `git commit -m "feat: 添加XX功能"`
4. **推送分支**: `git push origin feature/your-feature-name`
5. **创建PR**: 在GitHub上创建Pull Request

## 📝 Git工作流

### 分支策略
- `main` - 生产分支，仅包含稳定版本
- `develop` - 开发分支，日常开发基础
- `feature/*` - 功能分支，如 `feature/task-config-system`
- `bugfix/*` - 修复分支，如 `bugfix/theme-color-issue`
- `release/*` - 发布分支，如 `release/v1.0.0`

### 提交规范
```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 重构代码
test: 测试相关
chore: 构建配置
```
